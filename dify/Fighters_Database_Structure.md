# 🥊 ファイター1000人データベース構造

## 📋 データベース構造

### データ形式

各ファイターの情報を以下の形式で格納：

```markdown
## [ファイター名]

### 基本情報
- **国籍**: [国名]
- **階級**: [階級名]
- **スタイル**: [スタイル名]
- **年齢**: [年齢]
- **身長**: [身長]
- **体重**: [体重]

### 技術情報
- **得意技**: 
  - [技名1]: [詳細な説明]
  - [技名2]: [詳細な説明]
- **特徴的な動き**: 
  - [動きの説明1]
  - [動きの説明2]
- **試合での癖**: 
  - [癖の説明1]
  - [癖の説明2]
- **技術的な特徴**: 
  - [特徴1]
  - [特徴2]

### 実績
- **レジェンド度**: ★★★★★（5段階評価）
- **主な実績**: 
  - [実績1]
  - [実績2]
- **有名な試合**: 
  - [試合名]: [説明]

### トレーニング方法
- **練習スタイル**: [説明]
- **重視しているポイント**: [説明]
```

  - フットワークが非常に軽い。常に動き続ける。

- 重視しているポイント: フットワークとカウンターのタイミング。

## 📝 データベース作成のガイドライン

### 1. データの収集

以下の情報源からデータを収集：
- 公式サイト
- 試合動画
- インタビュー記事
- 専門家の分析

### 2. データの構造化

- 各ファイターの情報を統一された形式で記録
- 技術的な特徴を細かく記録
- 試合での癖を具体的に記録

### 3. データの検証

- 情報の正確性を確認
- 複数の情報源で検証
- 専門家の意見を参考にする

## 🔧 Difyナレッジスペースへのインポート

1. **Dify Console**にアクセス
2. **ナレッジスペース** → **新規作成**
3. **名前**: `Fighters_Database_1000`
4. **データのインポート**:
   - マークダウンファイルをアップロード
   - または、テキスト形式で直接入力

---

**重要**: データベースは継続的に更新し、最新の情報を維持してください。
