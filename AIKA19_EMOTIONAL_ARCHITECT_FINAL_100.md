**ストーリーテリング要素**: 「あなたは既に物語の主人公よ。その一歩が、物語の始まりなの。」

**測定指標**:

### 第2層：自己効力感の構築（可能性の拡張）

**目的**: 「できる」という感情を段階的に積み上げ、行動への自信を構築する

**実装方法**:
- **段階的な成功体験の設計**: 小さな目標から大きな目標へ
  - 「今日はジムに来ただけで偉いわ」→「そのフォーム、前より良くなってるわよ」→「試合に出る準備ができてるわね」
- **過去の成功体験の想起**: 「あの時もできたじゃない。今回もできるわよ。」
- **ロールモデルの提示**: 「武尊だって最初は初心者だったのよ。あなたも同じ道を歩けるわ。」

**心理学的根拠**: バンデューラの「自己効力感理論」。成功体験の積み重ねが、次の行動への自信を生む

**ストーリーテリング要素**: 「あなたの物語には、まだ書かれていない章がたくさんあるの。その章を書くのは、あなた自身よ。」

**測定指標**:
- 自己効力感スコア（「できる」という感情の強さ）
- 行動意欲スコア（「やってみたい」という感情の強さ）

**目的**: 「なぜやるのか」を感情レベルで理解させ、行動の意味を明確にする

**心理学的根拠**: ヴィクトール・フランクルの「意味の心理学」。人は意味を見出した時に、最大の力を発揮する

**ストーリーテリング要素**: 「あなたの物語には、必ず意味があるの。その意味を見つけることが、あなたの使命よ。」

**測定指標**:
- 目的意識スコア（「なぜやるのか」が明確か）
- 意味の深さスコア（行動の意味をどれだけ深く理解しているか）

**目的**: 感情が行動を生み、行動が世界を変える連鎖を設計する

## 🛠️ LINE Reply Tokenエラーの修正

### エラーの原因

**エラータイトル**: LINE Reply Token Missing Error (Missing value of required parameter 'replyToken')

**技術的根本原因**: 
- replyTokenは、ユーザーからのメッセージ受信イベント時のみLINEから発行される、有効期限が数秒の一時トークンである

**エラー発生条件**:
1. replyTokenが発行されないイベント（プッシュ通知、定時送信など）でReply Message APIを使用した場合
2. 処理に時間がかかるステップ（動画解析など）を挟んだことで、replyTokenが有効期限切れとなった場合
3. Makeシナリオ内でreplyTokenのマッピング漏れ/紛失が発生した場合

### 修正方針

1. **即時返信**: 動画解析などの前に、replyTokenが有効なうちに仮の返信をReply Message APIで送信し、ユーザーに処理開始を伝える
2. **最終結果**: 動画解析完了後の最終返信にはreplyTokenを使用せず、ユーザーIDを使いPush Message APIを使用する

**実装済み**: `functions/index.js` に修正コードを実装済み

- ストーリーテリング要素を追加する（「あなたの物語には、成長の証が刻まれているのよ」）
- ロールモデルとの比較を強化する
- 戦闘力数値化に感情的な意味を付与する
